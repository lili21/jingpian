# Jingpian

Jingpian 是一个面向品牌方、商家与内容团队的专业 landing page 项目，核心表达是：**先审分镜，再出样片**。

这个仓库当前承载的是 Jingpian 的首版官网 / 落地页原型，用于展示产品定位、交付方式、采购保障与专业版内测申请入口。

## 项目定位

Jingpian 不是一个“一键自动出爆款视频”的黑盒工具。

它强调的是更适合中国 B2B / 商业内容场景的工作方式：

- 先明确业务简报
- 先评审分镜结构
- 再进入样片生成与交付
- 通过更清晰的评审依据，降低方向误判、返工与跨团队沟通成本

当前页面主要服务于以下目标：

- 展示 Jingpian 的产品与服务定位
- 用专业、克制、可信的方式承接意向用户
- 收集专业版内测 / 咨询线索

## 当前仓库内容

项目为纯静态实现，包含：

- `index.html` — 页面结构与内容
- `styles.css` — 视觉样式与响应式布局
- `script.js` — 前端交互与表单逻辑
- `PROJECT_BRIEF.md` — 初始产品/页面需求说明

## 页面内容结构

当前 landing page 主要包含以下模块：

1. Hero 首屏
2. 流程概览
3. 决策对比（为什么不建议直接文生视频）
4. 交付样张 / 匿名案例格式
5. 采购保障
6. FAQ
7. 专业版内测申请表单

## 设计方向

该页面不是偏社交平台爆款风格，而是偏：

- 专业服务页
- 中国 B2B 商业采购语境
- 克制、可信、可采购
- 尽量减少“AI demo 感”

关键词包括：

- calm
- premium
- Chinese-first
- procurement-ready
- less AI demo, more business confidence

## 本地预览

在项目目录执行：

```bash
python3 -m http.server 8091
```

然后打开：

```text
http://127.0.0.1:8091
```

如果 8091 被占用，也可以换成其他端口，例如：

```bash
python3 -m http.server 8080
```

## 表单说明

当前版本的表单是演示用前端实现。

提交后会把数据保存在浏览器本地 `localStorage`，键名为：

```text
jingpian_professional_applications
```

这适用于：

- 演示提交流程
- 验证交互体验
- 本地预览与静态部署场景

当前**没有接真实后端或 CRM**。

## 部署建议

这是一个纯静态项目，适合部署到：

- Vercel
- Netlify
- Cloudflare Pages
- 任意支持静态文件托管的平台

如果使用 Vercel，通常可以直接在仓库根目录部署。
